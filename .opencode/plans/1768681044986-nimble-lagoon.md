# Plan: Add Exact Match Support for Routes

## Summary
Add per-route match type option. Routes can use `"contains"` (substring, current behavior) or `"exact"` (full URL match).

## New Config Format
```json
{
  "browser": "/Applications/Helium.app",
  "defaultProfile": "Default",
  "routes": [
    { "condition": "exact", "value": "https://app.example.com/", "profile": "Profile 1" },
    { "condition": "contains", "value": "github.com", "profile": "Profile 2" }
  ]
}
```

## Changes

### 1. Config.swift
Add `Condition` enum and update `Route` struct:

```swift
enum Condition: String, Codable {
    case contains
    case exact
}

struct Route: Codable {
    let condition: Condition
    let value: String
    let profile: String
}
```

### 2. Router.swift
Update matching logic to handle both types:

```swift
func route(url: URL) -> String {
    let urlString = url.absoluteString.lowercased()
    
    for route in config.routes {
        let value = route.value.lowercased()
        let matches = switch route.condition {
            case .contains: urlString.contains(value)
            case .exact: urlString == value
        }
        if matches {
            return route.profile
        }
    }
    
    return config.defaultProfile
}
```
